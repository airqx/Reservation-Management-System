# Reservation Management System

This document provides setup instructions, features, and operational guidelines for the Reservation Management System that was developed for the Resources Management department at KFUPM. This system eases up the booking process of rooms, includes classrooms, labs, or even other facilities, ensuring efficient use and excellent management.

## System Requirements

- Java JDK 11 or higher
- Android Studio, for Android development
- Git, for version control

## Usage Instructions

-Login: Start by logging in using your credentials. If you do not have credentials, please contact with the system administrator to be added 

- Make Reservations: Navigate to the reservation page and select the specific date, time, and room or facility you wish to book.

- View and Cancel Reservations: Go to the 'View Reservations' section to view your current reservations. Then, select any reservation to get it canceled if you want.

- Admin Functions: As admins you are easily able to view all reservations and cancel any of them in case you want to.

## Features:

- Direct Booking: Users can book facilities directly through the system without any kind of administrative intervention.

- Gender-Specific Booking Restrictions: Certain facilities are restricted by gender as this align with the university policies.

- Event-Based Reservations:  Users can create events for group activities, allowing others to join them.

- Administrative Oversight: Admins are able to  view, manage, and cancel reservations, if so email notifications would be sent upon all cancellations.
